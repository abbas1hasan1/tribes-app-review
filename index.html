Tribes UI/UX Review

App-Wide Themes

These are the recurring patterns we noticed across every screen in the app. Read this first, then dive into screen-specific tabs for details.

1. Visual Consistency — Icons & Colors

Home Match Chat Profile
  • Status bar green doesn't match header green (Home #1)
  • Filter icon style doesn't match nav icons (Match #3)
  • Status badges use off-brand colors (Chat #1)
  • Menu icons use mixed/inconsistent styles (Profile #1)
Adopt one icon family (e.g. Phosphor, Lucide) app-wide. Audit all color usage against brand palette.

2. Low Contrast & Flat Visual Design

Home Match Profile
  • Stats section background feels disconnected (Home #7)
  • Match screen background is washed out (Match #1)
  • Profile screen feels dull and flat (Profile #3)
  • Background card colors feel off in Seeking/Offering (Home #12)
Increase contrast between surfaces. Use brand dark green for section backgrounds, white for cards. Add depth with subtle shadows.

3. Remove Redundant Information

Home Chat Profile
  • "Welcome, Abbas Hasan!" greeting is unnecessary (Home #2)
  • Stats cards (All matches, Pending Wants) appear on both Home AND Profile (Home #5–6, Profile #2)
  • "Recent Partial Matched" section duplicates Match tab (Home #+)
  • Arrow icons on chat rows are redundant (Chat #2)
Each piece of data should have ONE canonical home. Remove duplicates.

4. Simplify Naming & Language

Home Directory
  • "All your WANTS" → "What You're Seeking" (Home #8)
  • "All your HAVES" → "What You're Offering" (Home #9)
  • "All matches" → "Matches" (Home #5)
  • "Pending Wants" → "Pending" (Home #6)
  • "My Directory" → "Listings" (Directory #13)
Use plain, natural language. Drop "All", "My", and quoted jargon.

5. Reduce Card & List Information Density

Home Directory Match
  • Home cards show too much — price, type, time, matched status (Home #10)
  • Directory list items show too much detail (Directory #16)
  • Match cards have redundant "Check Profile" link (Match #2)
Show only title + status indicator in lists/cards. Details on tap/drill-down only.

6. Rethink Layout Patterns

Home Directory Match Chat
  • Vertical card stack → horizontal scrollable grid (Home #11)
  • Seeking/Offering tabs → single scrollable page (Directory #14)
  • Match list → one-at-a-time focused view (Match #2)
  • Duplicate chat threads → one thread per person (Chat #3)
  • Center-align text in match cards (Match #4)
Each screen needs a layout that fits its purpose — scan (horizontal grid), focus (single card), or list (consolidated threads).

7. Missing Core Functionality

Home Directory
  • No way to delete listings (Directory #21)
  • No swipe-to-edit gesture (Directory #21)
  • Notification bell has no count badge (Home #4)
  • Points/coins badge visible but feature not ready (Home #3)
  • Close button overlaps nav bar (Directory #19)
  • Detail view lacks visual structure (Directory #20)
Prioritize delete/edit for listings. Hide unfinished features. Fix CTA overlap.

8. Quick Wins & Extras

Directory
  • Add stats summary cards at top of directory (Directory #15)
  • Add edit pencil icon on each row (Directory #17)
  • Keep FAB "+" button as-is (Directory #18)
Small additions that improve usability without major rework.

Home

Header area, stats cards, greeting text, coins badge, and notification bell.

1 2 3 4 5 6 7
Visual Consistency
1

Green color mismatch

There's a visible line between the system status bar and the app header — the two greens don't match. Ensure the status bar background uses the same #2D4A3E brand color.

Low Contrast
7

Change stats background to dark green

The current cream/yellow background for the stats section feels disconnected from the brand. Change it to #2D4A3E (same dark green as header/footer) and keep the cards white for contrast.

Redundant Info
2

Remove "Welcome, Abbas Hasan!"

Greeting text is redundant — the user knows who they are. Removing it frees up header space and feels less cluttered. Consider a simple "Home" title or just the logo.

+

Remove "Recent Partial Matched" section entirely

This section (visible when scrolling down) takes up a lot of space with a single match card showing distance, profiles, seeking/offering detail, "Read more" and "Send Offer" buttons. It's too much for the home screen — matches belong on the dedicated Match tab. Remove this section to keep the home page focused.

Naming & Language
5

Rename "All matches" → "Matches"

Drop "All" — it's redundant. Remove the "/3" fraction format and just show the count ("2"). Center-align the number and label within the card.

6

Rename "Pending Wants" → "Pending"

Simplify the label. Just show the number, centered within the card — same treatment as the matches card.

Missing Functionality
3

Hide coins / points badge (500)

Points system is not functional yet. Showing it sets an expectation that isn't met. Hide until the feature is ready to launch.

4

Notification bell — add numeric count

Keep the bell icon but replace the red dot with a numeric badge (e.g. "3") so users know how many unread notifications they have at a glance.

Home (Scrolled)

Seeking & Offering sections, card layout, and information density.

8 9 10 11 12
Low Contrast
12

Background card colors feel off

The muted sage/green tinted backgrounds on the Seeking and Offering sections feel disconnected from the rest of the app's styling. Noted for a future design pass — not a priority for this round, but flag for PJ.

Naming & Language
8

Rename "All your WANTS" → "What You're Seeking"

The current "All your 'WANTS'" label with quotes feels awkward and overly literal. Use friendlier, more natural language — something like "What You're Seeking" or "Your Requests". Drop the quotes and "All your" phrasing.

9

Rename "All your HAVES" → "What You're Offering"

Same treatment as Seeking — replace "All your 'HAVES'" with something natural like "What You're Offering" or "Your Services". Keep the language consistent between the two sections.

Info Density
10

Simplify cards — remove price & extra details

The cards currently show too much: price range, type, time ago, matched status all at once. Simplify each card to just the title (e.g. "Guitar Lessons") and a match indicator (matched/unmatched badge). Price, type, and other details should only appear when the user taps "See all" to go into the detail view.

Layout Patterns
11

Switch to horizontal scrollable grid

Currently only one card is visible at a time in a vertical stack. Switch to a horizontal scrollable grid that shows ~3 compact cards side by side. This lets users see more items at a glance and feels more modern. Each card should be a small tile — title + match badge only.

Directory

Tab-based "My Directory" screen with Seeking & Offering listings, card details, and FAB button.

13 14 15 16 17 18 21
Naming & Language
13

Rename "My Directory" → "Listings"

Simpler, more direct label. Users immediately understand this is their stuff — no need for the "My" prefix or the formal "Directory" wording.

Layout Patterns
14

Remove "Seeking"/"Offering" tabs

Show both sections on one scrollable page instead. The tab design hides half the content behind a tap — displaying both lets users scan everything at once without switching back and forth.

Info Density
16

Simplify list items

Remove price range, matched/unmatched badge, and "3 months ago" timestamp. Replace with: emoji icon + title + category/type (e.g. "🏠 Home Services · Service"). Add a "Manage All" link at the top of each section. Detail info should only appear when user taps into an item.

Missing Functionality
21

No way to delete listings — add swipe-to-delete and swipe-to-edit

  • No delete functionality — Users currently cannot remove listings they've created. This is a critical missing feature.
  • Add swipe gestures — Swipe left on a list row to reveal a delete action, swipe right to reveal an edit action. This is a standard mobile pattern (iOS/Android) that keeps the UI clean while providing quick access to destructive and edit actions.
Quick Wins
15

Add stats summary cards at top

Two cards: "Seeking" (count of user's listed seeking items) and "Offering" (count of user's listed offerings). Simple totals — e.g. "Seeking 10", "Offering 5".

17

Add edit pencil icon on each row

Clear affordance for quick editing without opening the full item detail view. A small pencil icon on the right side of each list row lets users jump straight to editing.

18

Keep FAB "+" button — no change needed

The floating action button works well for adding new seeking/offering items. Placement and behavior are solid — keep as-is.

Directory — Product Detail

Detail modal for individual listings — layout, CTA placement, and visual structure.

19 20
Missing Functionality
19

Close button overlaps Match nav icon

The "Close" CTA sits directly behind the floating Match circle in the bottom nav bar, making it nearly impossible to tap. Need to either raise the button above the nav or add bottom padding to the modal content so the CTA clears the nav area.

20

Detail view lacks visual structure and hierarchy

  • No cards, pills, or containers — "Type:Product | Category:Electronics" is unstyled raw text. Use pill/tag components for type and category, and wrap metadata in a light card or section container.
  • Cramped text with no hierarchy — Title, metadata, and description text have no breathing room. Needs more vertical padding between elements, a clearer font-size hierarchy (larger title, smaller meta text), and consistent left alignment.
  • No separation between list and detail — The detail modal slides up over the list with no clear boundary — the top of the modal blends into the list items above. Add a visible divider, top shadow, or rounded-top edge so the detail layer reads as a distinct surface.

Match List

Main match results screen — card layout, interactions, and visual consistency.

1 2 3 4
Visual Consistency
3

Filter icon doesn't match the rest of the app's icon style

The filter/sliders icon in the top-right has a different visual weight and style compared to icons in the nav bar and other screens. All icons across the app need to use a consistent icon set — same stroke weight, size, and style family — to feel cohesive.

Low Contrast
1

Low-contrast background makes the screen feel dull and flat

The light grey background behind the white match cards provides almost no contrast — the whole screen reads as washed out. Replace the grey with a soft green tint (consistent with Tribes' brand palette) so the white cards pop and the page feels more vibrant and intentional.

Info Density
2

Show one match at a time and simplify card interactions

  • One at a time — All 4 matches shown in a scrollable list feels like a directory re-skin, not a focused matching experience. Show one match at a time with swipe or next/prev navigation so each match feels intentional.
  • Remove "Check Profile" — The link next to the heart icon is redundant. Tapping the person's name should navigate to their profile directly — removes clutter and follows standard mobile patterns.
Layout Patterns
4

Center-align text within match cards

The WANT/HAVE labels, service names, and user info are all left-aligned, which feels unbalanced within the centered card layout. Center-align all text within the match cards for visual symmetry and a cleaner, more polished feel.

Chat List

Main chat inbox — conversation list, status badges, and navigation.

1 2 3
Visual Consistency
1

Status badges introduce off-brand colors

The "Pending" and "Accepted" badges use green tones that aren't part of the Tribes brand palette — these are new colors with no precedent in the design system. Use the established brand colors for all status indicators. Badge styling and row elements should also match patterns used in other screens for visual consistency across the app.

Redundant Info
2

Arrow icons are redundant — remove them

Every chat row has a → arrow icon that adds visual clutter. The entire row should be tappable (standard mobile pattern) — no need for an explicit navigation affordance. Remove the arrows to clean up the interface.

Layout Patterns
3

Consolidate duplicate conversations — one thread per person

The same person appears multiple times in the chat list (e.g., "Maisum Hashim" × 5, "Daniel Hudson" × 3). Conversations with the same person should be consolidated into a single thread, showing the most recent message. This is a known issue that has been previously discussed.

Profile

User profile and settings — account info, navigation, and support links.

1 2 3
Visual Consistency
1

Icons lack consistency — unify icon style across the app

The icons next to menu items (Profile Information, Wants, Haves, Past Projects, Help, Privacy Policy, Terms) use mixed styles that don't match each other or icons used elsewhere in the app. All icons should follow a single, consistent icon family that aligns with the rest of the app's visual language.

Low Contrast
3

Screen looks dull — needs better contrast and visual hierarchy

The profile screen feels flat and washed out. Light grey backgrounds, thin dividers, and muted text create a low-contrast appearance. Improve visual hierarchy with better contrast between sections, bolder typography for headings, and more intentional use of the brand color palette to make the screen feel polished and engaging.

Redundant Info
2

Remove redundant data cards — this info lives elsewhere

The "All matches" and "Pending Wants" summary cards duplicate information that already exists in other tabs (Match tab, Wants section). Remove these cards to eliminate redundancy — users can access this data from its primary location. Every piece of information should have one canonical home in the app.
